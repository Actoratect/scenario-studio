# 02. 追加提案機能

ご要望リスト＋ 01_concept.md の業界調査を踏まえ、現場で「これがあると別物になる」機能を提案します。
**P0/P1/P2** は MVP からの距離感の目安。

## A. 物語構造を支える機能

### A1. ビートシート/プロットテンプレート (P1)
- Save the Cat 15ビート、Hero's Journey 12ステージ、3幕構成、起承転結、Story Circle
- 新規プロジェクト時に「テンプレートから始める」を提供
- 各ビートに想定文字数、現在文字数、達成度バーを表示

### A2. 盛り上がりカーブ (Tension Curve) (P1)
- シーン/ビートに「緊張度 0–100」を設定
- 折れ線で時系列可視化
- 「3幕目の前に谷が必要」など教科書的セオリーをガイド表示

### A3. 伏線トラッカー (P1)
- 伏線を「設置 (planted)」「回収 (paid off)」のリレーションで管理
- 未回収伏線を警告
- 「回収シーンが伏線シーンより前にある」を検出

### A4. 視点 (POV) 管理 (P2)
- シーン/章ごとに視点キャラを設定
- POV 別の文字数集計、登場頻度ヒートマップ
- 「主人公が知りえない情報を地の文に書いていないか」AI チェック

### A5. ループ/分岐ストーリー対応 (P1)
- 「同じイベントの差分」をシーン Variant として持つ
- 周回ゲーや時間ループ物で同一イベントの差分を一覧

## B. キャラクター設計支援

### B1. 関係性マトリクス (P0)
- N×N マトリクスで全キャラ間の関係性を一覧編集
- セルに「関係種別 + 強度 0–10 + 一言メモ」
- グラフビューと双方向同期

### B2. キャラ口調シート (P1)
- 一人称、二人称、語尾、口癖、敬語レベル、感情ごとの差分
- 脚本執筆時に AI による口調一貫性チェック (P2)

### B3. キャラ年齢の自動計算 (P1)
- 「生年」と「物語の現在年」から年齢を導出
- Variant ごとに正しく出る (100年前なら 18歳、現代なら 118歳)

### B4. キャラアーク (P2)
- 「序盤の状態」→「転機」→「終盤の状態」を 3〜5 フレームで定義
- 各シーンが「どのキャラの成長点か」をタグ付け

### B5. 立ち絵/表情差分の管理 (P1)
- ノードに複数画像 (通常/笑顔/怒り/泣き) を登録
- 脚本中の `@happy` タグと連動してプレビュー表示
- ゲームへ立ち絵パスをエクスポート

## C. 世界観構築支援

### C1. 用語集 (Glossary) (P0)
- 造語/固有名詞/呼称ルールを一元管理
- 脚本中で自動検出・ハイライト
- 別の表記揺れを検出 (「アクトラテクト」「アクトラ」)

### C2. 地図ノード (P2)
- 舞台ノードに位置 (x,y) を設定し、地図画像上にピン表示
- 距離計算、ルート表示
- 地図画像を時代スライダで切替 (城が建つ前/後)

### C3. カレンダー/暦 (P1)
- 独自カレンダー定義 (1年=400日、4季×100日 等のファンタジー暦)
- グレゴリオ暦との対応表
- 月相/季節も時代軸で動く

### C4. ファクション (組織) ノード (P0)
- 「組織」型のノード。所属メンバー、敵対組織、本拠地、設立年/解散年
- グラフ上で枠 (Group) として表示

### C5. アイテム来歴 (P2)
- アイテムノードに「製造→所有者A→盗難→所有者B」の所有履歴
- 年表に重畳表示

## D. 執筆環境

### D1. 集中モード (P1)
- 全画面、サイドバー非表示、行間広め、タイプライタースクロール
- セッション目標 (語数/時間)、終わったらヒストグラムで達成度表示

### D2. スナップショット (P0)
- ノード単位、シーン単位、プロジェクト単位での「セーブポイント」
- Git とは別の軽量バージョニング (1ファイル単位の履歴を素早く差し戻し)

### D3. コメント/メンション (P1)
- 任意の行/ノードに付箋
- `@username` でメンション (Git author 名 or 設定ユーザ)
- 「Resolved」マーク

### D4. 改訂モード (Revisions) (P1)
- 「青稿/赤稿」のような色付き差分
- 校正者/監督が「ここ直して」マークして書き戻し

### D5. 読み上げ (TTS) (P2)
- 各キャラ別に音声 ID 割り当て、シーンを通しで読み上げ
- 校正、自然さ確認、所要時間測定
- Windows SAPI / macOS NSSpeechSynthesizer / OS 標準利用

### D6. コマンドパレット (P0)
- `Ctrl+Shift+P` で全機能/全ノードに即アクセス
- VS Code/Notion 風

### D7. リッチなクロスリンク (P0)
- `[[キャラ名]]` でリンク。Hover で要約ポップアップ
- 「どこからリンクされているか」逆引き

## E. 制作管理

### E1. ステータスとカンバン (P1)
- ノード/シーンに Status (`Draft/Review/Approved/Recorded/Locked`)
- カンバンビューでドラッグ移動

### E2. 進捗ダッシュボード (P1)
- プロット達成率、文字数推移、未翻訳数、未収録数
- グラフでチームに見せる用

### E3. Tagger / アセット要件抽出 (P1)
- 脚本中の `[SE: 雷鳴]` `[BG: 城内]` などを集計
- 必要美術アセット、SE、BGM、3D モデルのチェックリスト自動生成
- Unity 内のアセット紐付けまでできるとなお良い

### E4. 収録台本生成 (P1)
- キャラ別/ファイル別/章別で .docx, .pdf, .csv 出力
- Cue#, ファイル名, セリフ, 演技指示, 最大尺秒, 発音メモ

### E5. ロックとマージポリシー (P2)
- 編集中ノードを Git LFS lock or プロジェクト内擬似 lock
- マージ衝突を起こしやすい大規模ファイル (年表) を保護

## F. 整合性検査 (Linter)

### F0. 静的検査ルールの拡張可能性 (P1)
- ルール=C# クラスで追加可能
- 重要度: Error/Warning/Info
- ProjectSettings で ON/OFF

### F1. 参照整合性 (P0)
- 未定義キャラ参照、削除済みアセット参照
- リンク切れノード

### F2. 時系列矛盾 (P1)
- 「死亡後に登場」「未生まれで台詞」
- イベントの前後関係矛盾

### F3. 翻訳整合性 (P0)
- 未翻訳キー、孤児キー、プレースホルダ違反 (`{0}` の数違い)
- 文字数オーバー

### F4. キャラ口調 (P2, AI)
- LLM で口調・人称ブレ検出

### F5. 大事情報の登場順 (P2)
- 重要情報フラグ「秘密」「種明かし」を立て、誰がいつ知ったかを track

## G. ゲーム連携

### G1. Hot Reload (P1)
- エディタ実行中にデータ変更を即反映
- ScriptableObject の `RefreshFromYaml()` メソッド + AssetPostprocessor

### G2. Yarn / Ink Exporter (P1/P2)
- 既存 Yarn Spinner / Ink プロジェクトに合流できる出力
- 双方向は持たず Read-only エクスポート

### G3. 変数とフラグ (P0)
- `Variables` 名前空間。脚本から `set $met_alice = true` のように記述
- ゲーム側ランタイムで読み書き

### G4. プレビューランナー (P1)
- 脚本を擬似 ADV 風 UI で再生 (立ち絵+セリフ+選択肢)
- ゲーム実装前にライターが流れを確認

### G5. ローカライズパッケージ出力 (P0)
- Unity Localization パッケージ準拠の StringTable
- 独自 StringDB も並行対応

## H. AI 協調

### H1. インライン補完 (P1)
- セリフ書きかけで `Tab` 押下→ AI が続きを提案
- 当該キャラの口調シートをプロンプトに同梱

### H2. 整合性チェック (P1)
- 全ノード/全シーンを舐めて矛盾候補を列挙
- バックグラウンドジョブで実行、結果を Inspector に表示

### H3. 翻訳支援 (P0)
- 元文+コンテキスト (話者性別、口調、シーン要約、参考画像) を投げる
- TM (翻訳メモリ) 優先、なければ AI 翻訳

### H4. プロット壁打ち (P2)
- 「主人公の動機がない」「中盤に山場が薄い」などの構造批評
- AI Codex/Coding Agent が素のYAMLに対してパッチを出すワークフロー

### H5. 用語抽出 (P1)
- 既存テキストから「初出の固有名詞」を検出して用語集追加候補に

## I. アクセシビリティ/エルゴノミクス

- ハイコントラスト/ダーク/ライト/セピアテーマ
- フォントサイズ・行間・字間カスタマイズ
- 等幅/プロポーショナル切替
- 縦書きモード (脚本/小説で使う場合)
- ふりがな入力
- カラーフィルタ (赤緑色覚異常向け)

## J. 数値で押さえるスケール目標

| 指標 | 目標 |
|---|---|
| ノード数 | 〜10,000 |
| 1シナリオの行数 | 〜100,000 行 |
| 同時開きウィンドウ | 〜9 (3x3 グリッド) |
| グラフ描画ノード | 表示〜500 / 全体〜10,000 (LOD で間引き) |
| ファイル単位 | 1ノード=1ファイルが基本、シナリオは章ごとに分割 |
| 起動時間 | 1万ノードで 5 秒以内 (遅延ロード) |
| 自動保存 | 30 秒間隔、デバウンス |

## K. Phase 5+「ツール深掘り」追加候補

Phase 4 の 1.0 達成後、SaaS には行かずに **ツール自体を磨き続ける** 場合の追加候補。
詳細は `13_roadmap.md` Phase 5+。

### K1. AI 深掘り (P3)
- マルチプロバイダ抽象 (`LlmProvider` + `IAgentRunner`) 完成 — Claude API/Code、Codex、OpenAI、Gemini、Ollama、LM Studio、任意カスタム
- 「Inline 補完は Ollama、翻訳は Gemini、Agent は Claude Code」のような **用途別プロバイダ振分** UI
- ローカル LLM 標準サポート (Ollama 同梱推奨設定)
- マルチモーダル (画像→キャラ、音声→脚本、動画→演出)
- キャラ口調モデル (口調シートを LoRA 風に学習、当該キャラ専用の補完)
- 大規模整合性チェック (10万行)

### K2. 業界フォーマット完全対応 (P3)
- articy:draft XML 双方向、Final Draft FDX、Fountain 完全準拠、Yarn/Ink/Twee 双方向
- Trados / MemoQ XLIFF サブセット
- PDF テンプレート集 (映画/連ドラ/アニメ/ノベルゲー別)

### K3. 専門機能 (P3)
- 縦書きモード完成
- 吹き出しレイアウトプレビュー (漫画/ノベルゲー)
- 声優キャスティング
- 動的差分 (Branch Diff) — 「主人公が女性なら」など条件分岐の比較
- シミュレーション再生 (フラグ操作 + 自動 walkthrough)

### K4. プラグイン機構 (P3)
- 公式プラグイン API (TypeScript)
- 独自 Lens / Linter / Exporter / テンプレート
- サンプルプラグイン (ループもの、TRPG 用ルール 等)

### K5. Unity 統合の深さ (P3)
- Cinemachine / Animator / Live2D / VFX Graph / Timeline / Visual Scripting / Bolt 完全統合
- Unity Asset Store 公開

### K6. アクセシビリティ完成 (P3)
- WCAG 2.2 AA 準拠
- スクリーンリーダ完全対応
- UD フォント / ディスレクシア向け

### K7. パフォーマンス追求 (P3)
- グラフ 50,000 ノード対応
- 脚本 100 万行スクロール
- 起動時間 1 秒 (PWA 既開時)
