# 04. グラフエディタ (相関図)

## 目的

ノード間リレーションを視覚化し、その場で編集できるキャンバス。
**「相関図ツール」ではなく「グラフ上で物語の構築作業ができる」** ことを目指す。

## 1. 基本操作

| 操作 | 入力 | 既定動作 |
|---|---|---|
| パン | 中ドラッグ / Space + 左ドラッグ | キャンバス移動 |
| ズーム | マウスホイール / Ctrl+ホイール | カーソル基点ズーム (10%-400%) |
| 全体表示 | F | 選択ノードにフォーカス、無選択ならフィット |
| 矩形選択 | 左ドラッグ | 範囲内ノード選択 |
| 追加選択 | Shift+クリック | トグル |
| ノード移動 | 左ドラッグ | レイアウト固定 ON 時のみ |
| 詳細を開く | ダブルクリック | 「詳細パネル」or 「Inspector ウィンドウ」起動 |
| リレーション作成 | ノード辺をドラッグ→他ノード | 種別選択ポップアップ |
| ノード追加 | Tab / 右クリック→Add | テンプレ選択ポップアップ |
| 削除 | Del | ノード or リレーション |
| 複数編集 | 右ペインで一括フィールド編集 | Inspector が複数選択対応 |
| Mini-map ジャンプ | 右下 Mini-map クリック | カメラ移動 |
| 検索 | Ctrl+F | キャンバス上をフィルタ |

## 2. ビュー (Lens)

同じグラフでも目的別に「見え方」を切り替え可能。

| Lens | 表示内容 |
|---|---|
| **Relationship** (既定) | 親子/恋愛/敵対などソーシャルグラフ |
| **Faction** | ファクション枠で囲み、所属関係のみ |
| **Location** | 舞台ノードを地理的に配置 (地図モード) |
| **Timeline** | X軸=時代、Y軸=系譜の世代スイムレーン |
| **Plot Flow** | シーンノードの分岐フロー (articy Flow 風) |
| **Trope** | テーマ/モチーフ別のクラスタ |
| **Custom** | ユーザ定義 (フィルタ＋レイアウト保存) |

各 Lens は: フィルタ条件 (テンプレート/タグ) + リレーション種別の表示ON/OFF + レイアウト規則 を組み合わせ。

## 3. レイアウト

> **Phase 0 PoC-B 状況**: ベンチハーネスを `packages/frontend` の Bench Panel に実装済 (`pnpm -F frontend dev` → Graph Bench タブ)。同一合成グラフを SolidFlow (DOM、`solid-flow@1.0.4`) と Sigma.js (WebGL、`sigma@3` + `graphology`) で描画し、mount 時間と idle FPS を計測する。MVP/β/1.0 の各ライブラリ採用判断は、**実プロジェクト想定の N で本 harness を再実行** して根拠データを残す方針。`@xyflow/solid` (公式 SolidJS バインディング) は npm 未公開のため `solid-flow` (community 実装) を採用、登場時に再評価。

### 3.1 自動レイアウトアルゴリズム

| アルゴリズム | 適合 |
|---|---|
| Force-directed (D3 風) | 既定。緩やかな相関図向き |
| Hierarchical (Sugiyama) | 親子・所属の階層 |
| Circular | 派閥比較 |
| Grid | 大量ノードのカタログ的表示 |
| Radial | 主人公中心 |

### 3.2 手動レイアウトの保存

- 自動レイアウト後にユーザ調整した位置を Lens 単位で保存
- ノードの「ピン留め」(自動レイアウトに動かされない)
- リセットボタン

## 4. ノードのビジュアル

### 4.1 標準カード

```
┌─────────────────┐
│ ▣ Thumbnail     │  Status色の左帯 (Draft=灰, Approved=緑)
│ 太郎             │  displayName
│ Character | 18  │  templateId | age (era相対)
│ #protagonist    │  tags (上位3)
└─────────────────┘
```

### 4.2 LOD (Level of Detail)

ズームレベルでカード表現を切替:

- 100%以上: 完全カード (サムネ＋名前＋詳細)
- 50–100%: サムネ＋名前
- 25–50%: 色 dot ＋名前イニシャル
- 25%以下: 単色 dot のみ。クラスタは集合丸で表現

### 4.3 サムネイル

- ノードの thumbnail (なければテンプレ既定アイコン)
- 形状: 円/角丸四角 をテンプレ単位で選択
- Variant 切替時は当該 Variant の `thumbnailOverride` を優先

### 4.4 リレーション (エッジ)

- 種別ごとに色/破線/矢印形を変更 (`Relations/_types.yaml` の edgeStyle)
- 双方向は両端矢印
- 強度 (strength) で線の太さ
- ホバー: ツールチップで詳細
- 期間 (sinceEra→untilEra) が現在 Era 範囲外: 半透明

## 5. グラフ内編集

「グラフを開いたまま、シナリオ作業ができる」ことを重視。

### 5.1 サイドパネル (右ペイン)

選択ノード/エッジの詳細を編集する Inspector。
- フィールドフォーム (Template に従う)
- Notes (Markdown)
- 関連シーンへのジャンプ
- 関連リレーション一覧 (in/out)
- Variant 切替 (時代タブ)

### 5.2 ポップアップ編集

- ダブルクリック: フローティング詳細ウィンドウ (画面に複数開ける)
- 各ノードを「ピン留めウィンドウ」化、画面の任意位置にドック可能
- ピン留めは `07_window-system.md` のウィンドウシステムに準拠

### 5.3 シナリオ執筆へのジャンプ

- ノードを選択中に `S` キー: 関連するシーン一覧を Quick Open
- 「このキャラの初登場シーン」「直近の登場シーン」「未登場シーン候補」をワンタップ表示

### 5.4 グラフ→脚本のドラッグ&ドロップ

- ノードを脚本ペインにドラッグ → そのキャラのセリフ行が挿入される
- 「演出指示テンプレ」もドラッグで挿入可能

## 6. フィルタとサーチ

### 6.1 検索バー (上部)

```
[ 🔍 太郎 in:character tag:#protagonist era:modern ]
```

DSL 風:
- `in:<template>` 型で絞る
- `tag:#xxx` タグで絞る
- `era:<era>` 当該 Era に存在するノードのみ
- `from:<node>` 起点ノードからのリンク (距離指定 `from:tarou..2`)

### 6.2 強調モード

- マッチノードを彩色、非マッチは半透明グレー
- リレーションも同期

### 6.3 ハイライトプリセット

- 「主人公の交友関係」「悲劇の犠牲者だけ」などをユーザがプリセット保存

## 7. 大規模グラフへの対応

- ノード 10,000 / エッジ 50,000 を想定
- **仮想化レンダ**: 視野外は描画スキップ (Quad-tree spatial index)
- **クラスタ化**: 50ノード以上のグループは丸めて1ノード表示、ダブルクリックで展開
- **段階ロード**: 起動時はインデックスのみ、表示要求時にフィールド本体ロード

## 8. 履歴と Undo

- グラフ操作の Undo/Redo (移動、追加、削除、リレーション変更)
- データ変更は Unity の Undo システムに統合 (`Undo.RecordObject` 相当)
- ファイル直接編集 (外部) との競合: ファイル監視で再ロード+確認ダイアログ

## 9. 実装方針

### 9.1 採用技術

- **Unity UIToolkit GraphView** をベースに拡張
- 既存の ShaderGraph / VFXGraph と同系のUX
- ノード描画は VisualElement、Mini-map は GraphView 標準 + カスタム
- ヘビーなレンダ最適化が必要なら自前 RuntimeMesh で描画する選択肢を残す

### 9.2 データ→ビュー変換

```
Project (YAML)
   ↓ 起動時/監視で
GraphModel (in-memory, observable)
   ↓ Lens 適用
GraphViewModel
   ↓ レイアウト
GraphView (UIToolkit)
```

書き戻しは GraphModel に対して操作 → 自動でファイル保存 (デバウンス)

### 9.3 拡張ポイント

- 独自 Lens を C# クラスで追加可能 (`IGraphLens`)
- 独自レイアウトアルゴリズムも (`ILayoutAlgorithm`)
- ノード描画スキンも差し替え可能 (`INodeRenderer`)
