# 05. 時間軸・年表システム

## 設計の核

- **エポック (Era)** は時間軸上の「ラベル付き区間」または「点」
- 各ノードは Era ごとに **Variant** を持てる (03_data-model.md 参照)
- **Era スライダ**で UI 全体が一斉に切り替わる (グラフ・年表・脚本背景)
- **複数カレンダー**を共存可能 (グレゴリオ + 独自暦)

## 1. Era (エポック) の設計

### 1.1 種類

| 種類 | 例 | 用途 |
|---|---|---|
| 区間 (range) | 「ヴィクトリア朝 1837-1901」 | 文化・体制の時代区分 |
| 点 (instant) | 「現代」「2025年4月」 | スナップショット |
| 系統 (lineage) | 「主人公の幼少期」「壮年期」 | キャラ単位の時期 |

### 1.2 階層

Era は階層を持てる: 「中世」 ⊃ 「中世後期」 ⊃ 「ペスト流行期」

```yaml
- id: era.medieval
  label: 中世
  yearRange: [1000, 1500]
  children: [era.medieval_late, ...]
- id: era.medieval_late
  label: 中世後期
  yearRange: [1300, 1500]
  parent: era.medieval
```

### 1.3 並行 Era (パラレル)

- 同時代だが地域違い: 「日本: 戦国時代」と「ヨーロッパ: ルネサンス」
- ノードに `regionEras` 属性を持たせる
- 時間スライダ + 地域フィルタの両軸

## 2. Era スライダ UI

### 2.1 配置

- どのウィンドウからも見えるよう **グローバルツールバー** 上部に常駐
- スライダ＋ドロップダウン (定義済み Era から選ぶ) ＋年入力

### 2.2 操作

| 操作 | 動作 |
|---|---|
| スライド | 連続的に Era 移動 (連続再描画) |
| クリックスナップ | Era ラベル位置にスナップ |
| 矢印キー (`[` `]`) | 1 Era 単位で進む/戻る |
| Shift+矢印 | 1年単位 |
| Era ピン留め (P) | 「比較したい時代」を別ピンで保存、A/B 表示 |

### 2.3 A/B 比較モード

- 2つの Era をピン留めし、グラフを左右分割で同時表示
- ノード単位でも「Era A の値 / Era B の値」を並べて表示
- 「100年でこのキャラが何をしてきたか」を直感的に把握

## 3. 年表ビュー (Chronology)

### 3.1 表示形式

**ガントチャート + イベントピン**:

```
時代 →─────────────────────────────────────────────
           [Era: 創世期]──[Era: 古代]──[Era: 現代]
キャラ   ┌─────┐                ┌──────────┐
 太郎    │ 幼少 ─────── 老成 ──┘ (没)
 花子                ┌──── ─── ──── ──┘ (生存)
事件     ●大戦争       ●戴冠式       ●revolution
場所     ┃城建設 (1100)            ┃城焼失 (1900)
─────────────────────────────────────────────────
```

- 縦軸: ノード or カテゴリ
- 横軸: 時間 (Era 単位 / 年単位 切替)
- 行はドラッグで並べ替え、グルーピング可

### 3.2 表示できる要素

- **ノードのライフスパン**: 生年〜没年など、ノードの寿命イベント
- **イベントノード**: 戦争・即位・誕生・遺失などの一点
- **リレーションの期間**: 「友人関係 1850-1880」など
- **シーンノード**: シナリオの個別シーンが起きる年/era
- **ファクションの存続期間**

### 3.3 操作

- 時間軸ドラッグでズーム/パン
- イベントピンドラッグで年代変更
- ピン⇒詳細ジャンプ
- フィルタ: ノード型, タグ, ファクション
- レーン (swim lane) のグループ化保存

### 3.4 「現在 (Now) カーソル」

- スライダで選択中の Era 位置が縦線で表示
- グラフビューと完全同期

## 4. グラフ⇔年表⇔脚本の連動

選択や Era 変更が他ビューに即伝播。

```
Era スライダ
     ↓
   GraphView      ← Era Variant が反映される
   Timeline       ← Now カーソル移動、外れたイベント半透明
   Inspector      ← Variant タブが選択 Era にフォーカス
   Script         ← シーンの想定 Era と現在を比較警告
```

- 共通選択 (`SelectionContext`) を持ち、すべてのビューが購読

## 5. 年齢/期間の自動計算

### 5.1 ノード上の動的フィールド

```yaml
fields:
  birth_year: -50      # 物語現在=0 起点で 50 年前生まれ
```

`{tarou.age()}` のような式が脚本で使える:

| 式 | 意味 |
|---|---|
| `{tarou.age()}` | 現在 Era での年齢 |
| `{tarou.age(era.young)}` | 指定 Era での年齢 |
| `{events.war.yearsAgo()}` | 大戦争から何年前 |
| `{tarou.isAlive()}` | 当該 Era で生存しているか |

### 5.2 矛盾検出

- 「死亡 Era 以降のシーンに発話があれば警告」
- 「未生 Era のシーンに登場すれば警告」
- 死亡フラグは `lifeSpan: { birth: -50, death: 30 }` で表現

## 6. 暦 (Calendar) との関係

### 6.1 暦の役割

- 物語上の「日付」表現はカレンダー依存 (グレゴリオ/独自暦)
- 年表ビューは「絶対年 (内部)」と「カレンダー表記」を持つ
- カレンダー切替で表記が変わる (UI 側のみ。データは絶対年)

### 6.2 独自暦の例

```yaml
id: cal.eldoria
yearLength: 400
months:
  - { name: 春の月, days: 100 }
  - { name: 夏の月, days: 100 }
  - { name: 秋の月, days: 100 }
  - { name: 冬の月, days: 100 }
weekDays: [太, 月, 火, 水, 木, 金, 星]
epoch: { gregorianYear: 0, customYear: 1000 }
```

`1850-04-12 (Gregorian)` ↔ `エルドリア暦 2850 春の月 12日`

### 6.3 月相・季節

- 暦定義に「月相サイクル」「季節境界」を追加可能
- グラフの背景色を季節で変える (P2)

## 7. データ構造

### 7.1 Era ファイル

```yaml
# Eras/_index.yaml
calendar: cal.gregorian   # 既定カレンダー
eras:
  - id: era.creation
    label: { ja: 創世期 }
    yearRange: [-1000, -500]
    color: "#a0c8ff"
    ordinal: 0
  - id: era.modern
    label: { ja: 現代 }
    yearRange: [0, 0]
    color: "#80ff80"
    ordinal: 5
```

### 7.2 タイムライン保存状態

```yaml
# .editor/timeline_state.yaml  (Git ignore)
currentEraId: era.modern
pinnedEras: [era.young, era.modern]
viewLanesOrder: [character, faction, event]
filters:
  templateIds: [character, event]
  tags: [main]
```

## 8. パフォーマンス

- イベント 100,000 でも滑らかに動かす
- 時間軸の **段階 LOD** (zoom-out で年→世紀単位に集約)
- 仮想化 (画面外行は描画スキップ)
- 期間帯の連続描画は Quad-tree でカリング

## 9. アクセシビリティ

- 色弱対応: パターン (斜線/ドット) でも種別識別
- 数値読み上げ (年代をスクリーンリーダ向けに「西暦千八百五十年」)

## 10. 将来拡張アイデア

- **複数並行世界線** (パラレル/ループ物用にレーンを世界線別に分岐)
- **歴史 IF 分岐** (本編とは別の「もし○○が生きていたら」分岐線)
- **時間軸への AI 注釈** (「この期間は史実だと×× が起きていた」)
